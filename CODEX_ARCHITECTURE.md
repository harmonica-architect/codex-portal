**Its relationship to the field:**
The Wheel is the *still point* in the spiral — the axis around which frequency

**Accessing the Oversoul through the Wheel:**
The 6-phase breath (Inhale → Hold → Exhale → Stillness → Inhale → Hold)

breath in (expansion) → hold (compression) → breath out (contraction) →
stillness (integration). The Wheel holds this pattern in geometry so the
body and the field can entrain to it.

### Data Flow Between Tools

#### Wheel → Resonator
- **Breath phase state** flows from Wheel to Resonator
  (which phase is active → what frequency range to emphasize)
- **Personal sigil** (from Wheel profile) → Resonator default cycle suggestion
  (if your sigil favors Stillness, suggest 6-0-6)
- **Coherence score** from Resonator feeds back to Wheel glyph overlay
  (high coherence → brighter glyph glow)

#### Resonator → Wheel
- **Breath cycle** drives the Wheel's phase display timing
- **Field coherence** from multi-user sync adjusts the Wheel's sync indicator
  (in-sync users see their dots converge)

#### MatrixGlyph → Wheel
- **Saved sigil** → imported into Wheel profile as starting glyph set
  (up to 3 glyphs for the sigil)
- **Preferred layer** → Wheel defaults to that archetype on load

#### Wheel → MatrixGlyph
- **Current breath phase** → pre-highlights the corresponding row in MatrixGlyph
  (if you're in Exhale, the Breath row glows softly)
- **Glyph overlay** from Wheel → MatrixGlyph can "lock" that configuration

#### MatrixGlyph → Resonator
- **Saved frequency combination** → plays as a chord in Resonator
  (select 3 glyphs in MatrixGlyph → Resonator plays them as harmonics)
- **Layer archetype** → suggests which soundscape to activate
  (Elemental layer → 639 Hz emphasis; Tonal → full chord; Quasi → subtle 852 Hz)

#### Resonator → MatrixGlyph
- **Current coherence level** → suggests difficulty in MatrixGlyph selection
  (high coherence → allow complex multi-layer selections)
- **Adaptive level** → filters which layers are selectable
  (Level 1: only 3 layers; Level 4: all 12)

## VII. TECHNICAL CONVERGENCE — How to Build This

### Shared State Architecture

One `codex_state` object in `localStorage`:

```javascript
const CODEX_STATE = {
  sigil: ['△', '◇', '◁△▷'],
  cycle: [5, 5, 5],
  difficultyLevel: 1,
  coherence: 0,
  savedCombos: [],
  onboardingComplete: false,
  activeTab: 'wheel'
};
```

Each app reads/writes to this state. Tool-specific extensions live
in their own keys (`resonator_profile_[name]`, `matrix_saved_combos`).

### Server Extensions (server.js)

```javascript
// New message types for cross-tool sync:
{ type: 'sigil_update', sigil: ['△','◇','◁△▷'], userId }
{ type: 'coherence_update', coherence: 72, userId }
{ type: 'cycle_suggestion', cycle: [6,0,6], reason: 'coherence_70' }
{ type: 'combo_played', freqs: [432, 528, 639] }
```

### Future: Render Deploy (single unified app)

When all three are merged: `npm run build` produces a single
`dist/` folder deployed to Render. No more GitHub Pages split.
